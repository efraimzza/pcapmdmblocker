pm grant  com.emanuelef.remote_capture.debug android.permission.WRITE_SECURE_SETTINGS

dumpsys package com.emanuelef.remote_capture.debug | grep WRITE_SECURE_SETTINGS

שלב 1: הקמת ה"מחסן" ב-GitHub (גיטהאב)
פתיחת חשבון: היכנס לאתר GitHub.com ולחץ על Sign up (הרשמה). בחר שם משתמש (זה יהיה ה-Owner שלך), אימייל וסיסמה.
יצירת המחסן (Repository): * אחרי שנרשמת, לחץ על סמל ה"חתול" בפינה העליונה כדי להגיע לדף הבית.
חפש כפתור ירוק שכתוב עליו New (חדש) ליד המילה Repositories (מאגרים).
בשורת Repository name (שם המאגר), כתוב: mdmupdates.
רד למטה ולחץ על הכפתור הירוק Create repository (צור מאגר).
יצירת ה"מפתח" (Personal Access Token): זהו קוד סודי שמאפשר לדף הניהול שלך "לדבר" עם המחסן.
לחץ על תמונת הפרופיל שלך בפינה העליונה -> Settings (הגדרות).
בתפריט משמאל, רד הכי למטה ל-Developer settings (הגדרות מפתחים).
לחץ על Personal access tokens (טוקנים לגישה אישית) -> Tokens (classic) (טוקנים קלאסיים).
לחץ על Generate new token (צור טוקן חדש) -> Generate new token (classic).
בתיבת Note (הערה), כתוב: MyMDM.
חשוב מאוד: סמן ב-V את הריבוע הראשון שכתוב לידו repo (מאגר) ושכל תתי הסעיפים של repo יסומנו.
רד לסוף העמוד ולחץ על Generate token (צור טוקן).
העתק את הקוד: יופיע קוד שמתחיל ב-ghp_. תעתיק אותו עכשיו לפתק במחשב. ברגע שתסגור את הדף, הוא ייעלם!
שלב 2: הכנת ה"שלט רחוק" (קובץ הניהול)
עכשיו נגדיר את הקובץ admina.html שיהיה מרכז השליטה שלך.
פתיחת הדף: פתח את הקובץ admina.html בדפדפן (פשוט ללחוץ עליו פעמיים).
הגדרות ראשוניות: לחץ על הכפתור ⚙️ הגדרות חיבור.
מילוי הנתונים:
Owner (בעלים): שם המשתמש שלך ב-GitHub.
Repo (מאגר): mdmupdates.
Token (טוקן): הדבק את הקוד הארוך (ה-ghp_...) ששמרת מקודם.
לחץ על שמור הגדרות. המערכת מוכנה!
שלב 3: הגדרת המכשיר אצל הלקוח (פעם אחת בלבד)
כדי שהמכשיר של הלקוח ידע "להקשיב" לך, צריך לקשר אותו למחסן שלך.
פתח את אפליקציית ה-MDM במכשיר הלקוח.
לחץ על הלחצן השלישי (זהו לחצן ה-Backup/Settings - גיבוי/הגדרות).
הזן את הפרטים שלך:
Owner (בעלים): שם המשתמש שלך ב-GitHub.
Repo (מאגר): mdmupdates.
Client Name (שם לקוח): תן ללקוח שם (למשל moshe123).
שלב 4: איך מבצעים שינוי מרחוק? (התהליך היומיומי)
כאן קורה הקסם. נניח שאתה רוצה לחסום אפליקציה ללקוח moshe123.
הכנת ה"חבילה": בצע את כל השינויים באפליקציה אצלך במכשיר. בסיום, לחץ על "ייצוא הגדרות". ייווצר לך קובץ (בדרך כלל מסתיים ב-.zipmdm).
העלאה ל"ענן": פתח את דף הניהול (admina.html) אצלך:
שם לקוח: כתוב moshe123.
קוד עדכון (Update Code): המצא קוד כלשהו, נניח 5566. הקוד הזה הוא כמו שם של מגירה בתוך המחסן.
בחירת קובץ: בחר את הקובץ שיצרת בשלב 1.
לחץ על ⬆️ העלה קובץ.
מתן ה"אור הירוק": לחץ על הכפתור הירוק ✅ אשר (Ready=1). זה אומר למערכת: "הקובץ מוכן, אפשר להוריד".
הפעלה אצל הלקוח: אמור ללקוח ללחוץ על הלחצן הרביעי באפליקציה שלו ולהקיש את הקוד שנתת לו: 5566.
המכשיר שלו יוריד את הקובץ מה-GitHub שלך ויתעדכן מיד!
דברים שחשוב לדעת (האותיות הקטנות)
Ready=1 (מוכן): מאפשר ללקוח להוריד את העדכון.
Ready=0 (חסום): אם תלחץ על זה, גם אם הלקוח יקיש את הקוד הנכון, המערכת לא תיתן לו לעדכן. מומלץ ללחוץ על זה אחרי שהלקוח סיים את העדכון ליתר ביטחון.
שמות באנגלית: תמיד עדיף להשתמש באותיות באנגלית (למשל Client1 במקום לקוח1) כדי למנוע תקלות.
